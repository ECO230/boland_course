Coin Flip Simulator – Shiny App Context Document

Overview

This Shiny application demonstrates how human intuition about randomness differs from true stochastic processes.

The app simulates five sequences of coin flips: - Four sequences are generated using a true fair coin (Bernoulli with p = 0.5). - One sequence is generated by a human-like algorithm pretending to flip a coin.

The goal is for users to identify which sequence was generated by a human pretending versus real randomness.

Core Learning Objective

Humans underestimate how often long runs occur in truly random processes.

Real random processes: - Frequently produce long runs (e.g., HHHHHH). - Do not “self-correct”.

Humans pretending: - Avoid long streaks. - Over-alternate (HTHT patterns). - Try to “balance” outcomes.

This app visualizes that contrast.

App Structure

Single-file app.R Shiny app.

Required Header

The app begins with:

if (requireNamespace("renv", quietly = TRUE)) {
  renv::load("/data/junior/boland_course")
}

This ensures it loads the shared project environment when deployed.

UI Components

Controls

  • SelectInput for number of flips: 5, 10, 30, 50, 100 (default 50)
  • Checkbox toggle for highlighting runs (>= 3)
  • Simulate button
  • Reveal button (reveals which column is human)

Display

  • Five columns displayed side-by-side
  • Each column shows ordered flip outcomes (H or T)
  • One column is randomly assigned to be human-generated
  • Reveal adds a badge to identify the human column

Sequence Generation

Real Coin Flips

flip_real <- function(n) {
  sample(c("H", "T"), size = n, replace = TRUE)
}

Pure Bernoulli(0.5). Long streaks are allowed naturally.

Human Pretending Algorithm

The human generator is rule-based and psychologically informed.

Behavior rules:

  • Baseline tendency: alternate (HTHT-like)
  • Runs of 2: common
  • Runs of 3: occasional
  • Runs of 4: very rare
  • If a run of 4 ever occurs, no additional runs >= 4 allowed

Switch probability depends on current run length:

Run Length Switch Probability
1 ~0.65
2 ~0.78
3 ~0.97
>=4 forced switch

This produces realistic human bias while still looking somewhat random.

Run Detection

Run detection uses rle():

run_lengths <- function(x) {
  r <- rle(x)
  rep(r$lengths, r$lengths)
}

Each position is labeled with the length of the run it belongs to.

Highlighting Logic

Highlighting is optional (checkbox).

Rules: - Runs of 3+ are highlighted - Longer runs get stronger shading - Implemented via CSS classes: - run3 - run4 - run5 - run6+

No alternating row banding (removed for clarity).

Reveal Mechanism

  • On simulate: one of five columns randomly assigned as human
  • Reveal button toggles a badge showing:
    • “Human (pretending)”
    • “Real flips”

Before reveal, all columns are labeled simply “Coin Flip”.

Architectural Notes

  • No animation (static render for stability)
  • Pure base Shiny (no additional JS dependencies)
  • CSS injected via tags$style
  • Uses reactiveValues to store:
    • sequences
    • human column index
    • reveal state

Teaching Value

This app supports classroom demonstrations of:

  • Law of Large Numbers misconceptions
  • Gambler’s fallacy
  • Human randomness bias
  • Run length expectations in Bernoulli trials

It works particularly well before introducing formal probability theory.
